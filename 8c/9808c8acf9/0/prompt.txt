## Implementation Details

### Commit Message Generation
Use Claude CLI to generate conventional commit message from staged diff:
```bash
echo "<diff>" | claude -p "Generate conventional commit message" --model claude-haiku-4-5-20251001
```
Fallback to pattern-based message if Claude fails.

### PR Creation Flow
1. Check `git status --porcelain` for changes
2. If changes: stage, generate message, commit
3. Call `processIntents()` from intent.ts (already exported)
4. Check for remote tracking, push with `-u` if needed
5. Check `getCurrentPR()`, create via `gh pr create --fill` if null
6. Call `reviewHandler()` with `{ skill: "vibereq:code-review" }`
7. Output PR URL

### Address PR Flow
1. Get PR number via `getCurrentPR()` or `--pr` option
2. Query unresolved threads with enhanced GraphQL (include comment details)
3. Get diff via `git diff main...HEAD`
4. Format output with sections:
   - Unresolved comments (file:line, author, body)
   - Current diff
   - Agent exploration instructions

### Error Handling
- No changes to commit: Skip commit step, continue
- Intent generation fails: Log warning, continue
- PR already exists: Use existing (don't fail)
- Review fails: Log error, still return PR link
